Un item passe à validé après accord d'au moins un expert avec la position du catalogue ; il passe à contesté dès qu'apparaît un « Ça dépend », un désaccord entre experts, une réponse contraire au catalogue ou un manque signalé. Une validation acquise sur une version antérieure du catalogue est comptée obsolète, jamais comme encore valide. Les items contestés forment l'ordre du jour d'adjudication (report_<campagne>.md). La revue des missions candidates MC-034–066 réutilisera ce dispositif.

Cette page ne publie que des agrégats : les noms des experts, leurs réponses libres et les commentaires d'adjudication restent dans des fichiers locaux ignorés par Git. Les indicateurs sont aussi disponibles en JSON.

Règle de continuité. L'ontologie et les catalogues portent la sémantique métier ; le State Model normalise le vocabulaire dynamique ; le Method Catalog porte les décompositions. Le HDDL est une projection exécutable — jamais une source métier indépendante.
